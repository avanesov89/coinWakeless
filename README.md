# Учёт дропа клана — онлайн-версия

Веб-страница `index.html`, которая работает у вас на любом устройстве по ссылке. Данные (игроки, походы, дроп, расчёты) хранятся в бесплатной базе Firebase и синхронизируются в реальном времени.

## Шаг 1. Создать базу данных (Firebase, бесплатно)

1. Откройте [console.firebase.google.com](https://console.firebase.google.com) и войдите под Google-аккаунтом.
2. **Add project** → введите любое имя (например, `wakeless-loot`) → можно отключить Google Analytics → **Create project**.
3. В левом меню: **Build → Firestore Database → Create database**. Выберите любой регион, режим — **Start in test mode** (правила ниже сразу их ужесточат).
4. В левом меню: **Build → Authentication → Get started → Sign-in method → Anonymous → Enable → Save**. Это нужно, чтобы приложение могло читать/писать данные без полноценной регистрации пользователей.
5. Нажмите на шестерёнку рядом с "Project Overview" → **Project settings**. Внизу раздела "Your apps" нажмите иконку `</>` (Web) → задайте имя → **Register app**. Появится блок кода с объектом `firebaseConfig` — скопируйте его целиком (от `{` до `}`).

## Шаг 2. Настроить правила безопасности Firestore

В Firestore Database → вкладка **Rules** — замените содержимое на:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

Нажмите **Publish**.

> Важно понимать: страница на GitHub Pages публичная, конфиг Firebase виден в исходном коде страницы любому, кто её откроет. Правило выше требует только "быть авторизованным" — а анонимная авторизация происходит автоматически для всех посетителей сайта. То есть это не защита от целенаправленного взлома, а барьер от случайных людей. Для этого в приложении есть отдельный пароль-заглушка (задаётся при первом входе). Не используйте этот способ для действительно чувствительных данных.

## Шаг 3. Выложить страницу на GitHub Pages

1. Зайдите на [github.com](https://github.com), создайте новый репозиторий (**New repository**), например `clan-loot`. Можно Public или Private — Pages из Private репозитория бесплатно работает только на аккаунтах GitHub Pro/Team/Enterprise, поэтому для бесплатного варианта выбирайте Public.
2. Загрузите в него файл `index.html` (кнопка **Add file → Upload files**, перетащите файл, **Commit changes**).
3. Зайдите в **Settings → Pages**. В разделе "Build and deployment" выберите Source: **Deploy from a branch**, Branch: **main** / папка **/(root)** → **Save**.
4. Через 1–2 минуты сайт будет доступен по ссылке вида `https://<ваш-логин>.github.io/clan-loot/`.

## Шаг 4. Первый запуск

1. Откройте ссылку на сайт.
2. Вставьте скопированный на шаге 1.5 объект `firebaseConfig` в поле настройки — жмите "Подключить".
3. Приложение попросит задать пароль клана — придумайте и сохраните. Дальше и вы, и все, кому дадите ссылку и пароль, будете видеть одни и те же актуальные данные с любого устройства.
4. Конфиг Firebase сохраняется в браузере автоматически при следующих визитах — вставлять его повторно не нужно (кроме как на новом устройстве/браузере, тогда шаг 2 повторяется, а пароль уже будет общий).

## Как это устроено

- **Firestore** — облачная база данных Google, бесплатный тариф с запасом на клановые объёмы (десятки тысяч операций в день бесплатно).
- Изменения (добавили поход, отметили оплату) видны всем открытым вкладкам почти мгновенно.
- Логика расчёта долей: дроп, привязанный к конкретному походу, делится поровну между участниками именно этого похода; непривязанный дроп — поровну между всеми отмеченными в расчёте походами. Доля игрока — сумма его долей по тем дням, когда он реально ходил. Сумма всех долей всегда равна общей распределяемой сумме.
- Пароль клана хранится в самой базе (Firestore, документ `meta/app`) и его можно сменить в приложении на вкладке "Настройки".

## Если что-то пошло не так

- **"Не удалось подключиться" / ошибка авторизации** — проверьте, что в Firebase включена Anonymous-авторизация (шаг 1.4).
- **"Ошибка доступа к базе"** — проверьте правила Firestore (шаг 2), нажали ли Publish.
- Хотите начать с чистого листа или подключить другой проект Firebase — на вкладке "Настройки" есть кнопка "Отвязать это устройство от базы".
